## 已完成

### Phase 0（规划）: 项目协调与文档地基（✅ 2026-06-20）
**目标**: 作为协调者，完成项目规划——目录结构、复杂度评估、协作方案、接口契约草案、预研计划。
**关键发现**:
- 本项目不适用传统「前端/后端/数据库」三分；按 AI Agent 架构层切分为 5 条工作流（数据检索 / 编排核心 / 工具持久化 / API-UI / 评估）更自然。
- 最高风险集中在 LangGraph 四件套（subgraph/checkpointer/interrupt/store）与 Qdrant 混合检索，需预研先行。
- 三层持久化（Checkpointer/Store/PostgreSQL）的职责边界是最容易混乱的设计点，已在契约中显式约束。
**产出**:
- 目录结构（context/doc/memory/templates/src）
- `README.md`、`context/业务需求与项目上下文.md`
- `doc/技术实施方案.md`（含 6 张 mermaid 架构/流程图）
- `doc/AI协作分工方案.md`（复杂度评估 + 5 工作流 + 甘特 + 同步点）
- `doc/模块接口契约.md`（C0-C7 草案，待 Lock）
- `doc/技术预研计划.md`（4 个 spike）
- `memory/全局纪要.md`、`templates/*.yaml`
